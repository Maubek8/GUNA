Reglamentación Técnica – KIS/2025

(Aplicable a la Ley de Activos Virtuales y VASPs)

1. Formatos Estándar de Reportes

1.1 Reporte de Operación Sospechosa (ROS-KIS-01)

  • Identificador único del reporte
  • Datos básicos del reportante: VASP, clase, responsable de cumplimiento
  • Datos del cliente (nivel de KYC alcanzado)
  • Tipo de operación: intercambio, custodia, transferencia
  • Monto y fecha
  • Descripción de la sospecha (máx. 300 palabras)
  • Acciones tomadas: congelamiento, advertencia, monitoreo reforzado
  • Firma digital del oficial de cumplimiento

Formato de envío: JSON + firma digital (OpenPGP o X.509)

Plazo: 72h desde detección

Canal: Portal seguro KIS o API REST

1.2 Reporte de Operaciones en Efectivo/Fiat (ROE-KIS-02)

  • ID del reporte
  • Cliente (N1/N2)
  • Monto en fiat y moneda
  • Fecha y medio (transferencia bancaria, cash-in/out)
  • Destino u origen declarado
  • Validación documental (si aplica)

Umbral: ≥ USD 10.000 en 24h

Plazo: 7 días

Formato: CSV firmado o API

1.3 Reporte de Incidente Cibernético (INC-KIS-03)

  • Fecha y hora de detección
  • Tipo de incidente: hackeo, fuga de datos, caída prolongada, manipulación interna
  • Impacto: fondos, datos, disponibilidad
  • Medidas inmediatas tomadas
  • Estado actual
  • Plan de recuperación

Plazo: 72h desde detección

Canal: API cifrada + notificación por correo seguro

2. Especificación Travel Rule Light (TRL-KIS-2025)

2.1 Datos mínimos obligatorios

Ordenante:

  • Nombre completo
  • Identificador único (doc, wallet verificada, ID KIS)
  • País de residencia

Beneficiario:

  • Nombre completo
  • Identificador único
  • País de residencia

2.2 Umbral

Aplicable a transferencias ≥ USD 1.000 entre VASPs

2.3 Protocolos aceptados

  • Mensajería cifrada fuera de cadena (ej. Signal, Matrix, correo PGP)
  • API REST segura con TLS 1.3
  • Batch file diario cifrado y firmado (para VASP Clase A)

2.4 Respuesta ante fallos

Si no es posible obtener datos, el VASP debe:

  • a) Marcar la operación de riesgo alto
  • b) Ejecutar monitoreo reforzado
  • c) Reportar a KIS si hay sospecha

3. APIs y Endpoints Mínimos

/api/report/ros → POST ROS-KIS-01 /api/report/roe → POST ROE-KIS-02 /api/report/inc → POST INC-KIS-03 /api/travelrule/send → intercambio de datos ordenante/beneficiario /api/travelrule/ack → confirmación de recepción

Formato preferido: JSON

Cifrado: TLS 1.3 + JWT firmado

Autenticación: API Key + firma digital KIS

4. Plantillas Básicas (Modelo Light)

  • Manual PLD/FT: 2 páginas → umbrales, ROS, TRL
  • Plan de Ciberseguridad: 2 páginas → MFA, gestión de llaves, backups
  • Disclosure al Usuario: 1 página → tarifas, riesgos, plazos de retiro

5. Matriz de Cumplimiento Esencial (MCE)

  • Clase A: cumplimiento simplificado → checklists y ROS
  • Clase B: cumplimiento intermedio → ROS, ROE, TRL básico, auditoría anual
  • Clase C: cumplimiento pleno → ROS, ROE, TRL completo, pruebas de reservas trimestrales

6. Sandbox y Pruebas

Sandbox KIS acepta:

  • Simulación de ROS/ROE/INC con datos ficticios
  • Pruebas de Travel Rule entre VASPs con wallets demo

Duración: 12 meses, prorrogable
